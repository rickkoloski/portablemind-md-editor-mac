## 0. Approach

Three phases, smaller than D23. Most of the patterns are reused from D23 (connector method → API client → sheet/dialog → harness action → smoke).

1. **Connector + API client surface.** Add deleteFile / createDirectory / deleteDirectory through the layers (protocol, defaults, LocalConnector, PortableMindAPIClient, PortableMindConnector).
2. **UI.** Confirmation NSAlerts for the two delete operations; CreateDirectorySheet for create. Context-menu wiring. Cascade-close open tabs. Sidebar tree splice (closes the lingering `TODO-D23-tree-splice` for create + delete + the earlier saveAs/rename/move surfaces as a bonus).
3. **Harness + self-cleaning smoke.** Three new actions: pm_delete_file, pm_create_directory, pm_delete_directory. Smoke recipe demonstrates self-cleaning: create scratch dir → run D23 file ops inside → delete scratch dir → no residue.

D23 phase 6 (manual test plan + COMPLETE + roadmap + tag v0.7) is held until D23.1 ships so the test plan can use self-cleaning fixtures. D23.1's close-out folds into D23 phase 6.

## Phase 1 — Connector + API client surface

**Files updated:**

- `Sources/Connectors/Connector.swift`:
  ```swift
  func deleteFile(_ node: ConnectorNode) async throws
  func createDirectory(in parent: ConnectorNode,
                       name: String) async throws -> ConnectorNode
  func deleteDirectory(_ node: ConnectorNode) async throws
  ```
  Default implementations throw `.unsupported`.

- `Sources/Connectors/LocalConnector.swift`:
  - `deleteFile`: `FileManager.removeItem(at:)`. Trash semantics not provided in v1.
  - `createDirectory`: `FileManager.createDirectory(at:withIntermediateDirectories: false)`.
  - `deleteDirectory`: `FileManager.removeItem(at:)` (recursive by default).

- `Sources/Connectors/PortableMind/PortableMindAPIClient.swift`:
  ```swift
  func deleteFile(fileID: Int) async throws
  func createDirectory(parentPath: String, name: String)
       async throws -> DirectoryDTO
  func deleteDirectory(directoryID: Int) async throws
  ```
  All three reuse the existing `setAuthHeaders` helper. Response decoding: create returns `{success: true, llm_directory: {...}}`; delete returns `{success: true}`.

- `Sources/Connectors/PortableMind/PortableMindConnector.swift`:
  - Implements all three. `createDirectory` builds a ConnectorNode from the returned DirectoryDTO (matching the existing `tenantInfo(from: DirectoryDTO)` helper). Both delete methods parse the LlmFile/LlmDirectory id out of `node.id` (`"<connectorID>:dir:<id>"` for directories — confirm the prefix matches what D18 uses).

**DOD:**
- Build clean.
- Unit tests still pass (no behavior change to existing paths).
- Connector method spike — call deleteDirectory on a manually-created PM directory via harness; confirm server returns 200 and the directory is gone.

## Phase 3 — Harness + self-cleaning smoke

**Goal:** make the smoke pattern self-cleaning end-to-end. Closes the meta-debt that every D23 phase ran on real tenant data and required manual cleanup.

**Files updated:**

- `Sources/Debug/HarnessCommandPoller.swift`:
  - `pm_delete_file {nodeID, path}` — delete file by id; result file emits {ok}.
  - `pm_create_directory {parentNodeID, name, path}` — create directory; result file emits new node id + path.
  - `pm_delete_directory {nodeID, path}` — delete directory + cascade; result file emits {ok}.

**Self-cleaning smoke recipe (will live in the manual test plan):**

```bash
TS=$(date +%s) ; SCRATCH="d23-smoke-scratch-$TS"
# 1. Create scratch dir under /
echo "{\"action\":\"pm_create_directory\",\"parentNodeID\":\"portablemind:dir:ROOT\",\"name\":\"$SCRATCH\",\"path\":\"/tmp/r.json\"}" \
    > /tmp/cmd.tmp && mv /tmp/cmd.tmp /tmp/mdeditor-command.json
sleep 1
SCRATCH_ID=$(jq -r .newNodeID /tmp/r.json)

# 2. Run the D23 file CRUD smoke inside the scratch dir.
#    pm_save_as / pm_new_file / pm_rename / pm_move all parented to $SCRATCH_ID.

# 3. Cleanup: delete the scratch dir (cascade — files inside go too).
echo "{\"action\":\"pm_delete_directory\",\"nodeID\":\"$SCRATCH_ID\",\"path\":\"/tmp/r.json\"}" \
    > /tmp/cmd.tmp && mv /tmp/cmd.tmp /tmp/mdeditor-command.json
```

**DOD:**
- Self-cleaning smoke runs end-to-end: scratch dir created, file ops succeed inside it, scratch dir deleted, PM tenant has no residue.
- All harness actions emit a stable result file shape.
- D23 phase 6's manual test plan can adopt this recipe verbatim.

**Commit:** `D23.1 phase 3 — Harness actions + self-cleaning smoke recipe`

## Risks / open implementation questions

1. **Cascade tab close edge cases** (Q4). Path prefix matching needs to be precise — `/projects-old` shouldn't match `/projects`. Use `path == deletedDir.path || path.hasPrefix(deletedDir.path + "/")`.
2. **Tree splice for sync-children connectors** (Local). Q3 mitigates by scoping splice to PortableMind only in v1. Local refreshes via `FolderTreeWatcher` for filesystem changes — D6 behavior preserved.
3. **Permissions on delete.** Like D19 saveFile, delete may 401/403. The existing `.writeForbidden` ConnectorError variant maps to a "Write denied" message in the confirmation flow.
4. **Child count in delete-directory confirmation** (risk #4 in spec). v1 reads from already-loaded counts where available; falls back to a quick `GET /api/v1/llm_files?directory_path=...&limit=-1` if unloaded. The count is informational only — the operation runs regardless.
