## Phase-by-phase guidance

### Phase 1 — Connector + API client

The model_api side is already verified. Endpoints:
- `POST /api/v1/llm_directories` — create (body: `{llm_directory: {name, parent_path}}`).
- `DELETE /api/v1/llm_directories/:id` — destroy with cascade.
- `DELETE /api/v1/llm_files/:id` — destroy.

PortableMindAPIClient already has `setAuthHeaders` and `sendForLlmFile` from D23 phase 1 — reuse them. For `createDirectory`, add a parallel `sendForLlmDirectory` that decodes a `LlmDirectoryShowResponse` envelope. For the delete methods, no decode needed — just check the status code and body for `{success: true}` or fall through to the standard error mapping.

### Phase 2 — UI

Three patterns to compose:

1. **Confirmation NSAlert** for both deletes. Mirror the existing alert pattern from `MdEditorApp.saveAsFocused` (pre-D23 PM branch) — `NSAlert.runModal()` returns the button index. Don't use `beginSheetModal` (we're not blocking another runloop here).

2. **CreateDirectorySheet** — copy RenameSheet, simplify (no "different from current name" check; just non-empty + no `/`). Same accessibilityIdentifier scheme.

3. **Tree splice on ConnectorTreeViewModel.** Add `upsertNode(_ node: ConnectorNode, parentPath: String)` and `removeNode(id: String, parentPath: String)` methods that mutate `asyncChildren[parentPath]` in place. PortableMind connector only in v1 (Local refreshes via FolderTreeWatcher; Q3 in spec). Wire saveAs / newFile / rename / move / createDirectory / deleteFile / deleteDirectory all through these splice helpers in PMFileOperations — closes the `TODO-D23-tree-splice` debt.

Cascade-close open tabs is in PMFileOperations.delete:
```swift
// File delete: close any tab with matching connectorNode.id
// Directory delete: close any tab whose path is the deleted dir's
//   path OR has it as a path-component prefix (`path + "/"`)
```

### Phase 3 — Harness + self-cleaning smoke

Three new actions following the existing pattern. Each calls into PMFileOperations via the same code path the UI uses.

The self-cleaning smoke recipe in the plan §Phase 3 is the canonical pattern. The D23 phase 6 manual test plan (next deliverable) adopts it.
