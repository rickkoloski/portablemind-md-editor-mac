## Implementation Details

### What Was Built

- **Connector protocol** gains `deleteFile`, `createDirectory`, `deleteDirectory` with `.unsupported` defaults.
- **PortableMindAPIClient** gains:
  - `deleteFile(fileID:)` — `DELETE /api/v1/llm_files/:id`
  - `createDirectory(parentPath:name:)` — `POST /api/v1/llm_directories` with `{llm_directory: {name, path}}` (path is the full target — see deviations §1).
  - `deleteDirectory(directoryID:)` — `DELETE /api/v1/llm_directories/:id?cascade=true` (cascade always — see deviations §2).
  - Two new private helpers: `sendForLlmDirectory(_:)` and `sendForGenericSuccess(_:)`.
- **LocalConnector** implements all three via `FileManager`. NSFileNoSuchFileError → `.server(404)` for "already deleted" semantics.
- **PortableMindConnector** implements all three via the API client; private `directoryNode(from: DirectoryDTO)` builder + static `directoryID(from:connectorID:)` parser.
- **`ConnectorTreeViewModel`** gains `upsertNode(_:parentPath:)` and `removeNode(id:parentPath:)` splice mutators. `removeNode` also evicts cached `asyncChildren` entries under the deleted node's path (trailing-/ boundary so `/foo` doesn't match `/foobar`).
- **`PMFileOperations`** gains:
  - `delete(node:store:)` — hard delete with cascade-close (`closeTabsForFile` for files; `closeTabsInDirectory` for dirs with trailing-/ boundary check, Q4).
  - `createDirectory(in:name:store:)` — creates + splices into tree.
  - **Retrofitted** the existing D23 `saveAs`/`newFile`/`rename`/`move` paths to splice the cached tree (closes `TODO-D23-tree-splice`). `saveAs` gained a store-aware overload; the older signature stays for compat.
- **`CreateDirectorySheet`** (new) — variant of RenameSheet (single TextField + Save/Cancel + inline error).
- **`WorkspaceStore`** gains `createDirectoryRequest` payload + `requestCreateDirectory(in:)` / `dismissCreateDirectory()`.
- **Sidebar context menu** (`ConnectorTreeView`):
  - PM file rows: `Delete…` item with NSAlert confirmation ("Delete '`<name>`'? This can't be undone.").
  - PM directory rows: `New Folder…` and `Delete…` items. Delete excluded on root row (`level == 0`). Directory delete alert surfaces a child-count when known ("This will also delete N item(s) inside.").
- **Harness actions:** `pm_delete_file`, `pm_create_directory`, `pm_delete_directory`. All route through `PMFileOperations` (single code path with the UI).
- **AccessibilityIdentifiers:** `folderTreeRowDelete(id:)`, `folderTreeRowNewFolder(id:)`.

## Deviations from Spec

### 1. createDirectory request body — `path` field needed

Initial implementation sent `{llm_directory: {name, parent_path}}` per the spec's plan. Server returned 422 because the LlmDirectory model's `validates :path, presence: true` fires BEFORE the `before_validation :set_parent_path` callback that auto-derives `parent_path`/`depth` from `path`. Surfaced live during phase 2+3 smoke; fixed inline. Body now sends `{llm_directory: {name, path}}` where `path` is the full target (e.g. `/foo/bar`). Empty parent (`rootNode.path == ""`) and `"/"` both map to root → child path is `/<name>`.

### 2. deleteDirectory needs `?cascade=true`

Server returns 422 with "Cannot delete non-empty directory without cascade option" without an explicit cascade flag. Our UX collects user confirmation via NSAlert (with child-count surfaced), so the API client always passes `cascade=true`. Surfaced live during phase 2+3 smoke; fixed inline.

### 3. Phase 4 close-out folded into D23 phase 6

Plan §Phase 4 said D23.1's close-out would fold into D23 phase 6. Done — single manual test plan covers both deliverables; this COMPLETE doc cross-references the master D23 close-out.

### 4. Tree splice closed `TODO-D23-tree-splice`

Plan §Phase 2 described tree splice for create + delete. Bonus: retrofitted the existing D23 saveAs/newFile/rename/move paths to splice too, closing the longstanding debt that the sidebar didn't refresh after D23 ops. Net effect at v0.7: every PM mutation reflects in the sidebar without manual reload.
