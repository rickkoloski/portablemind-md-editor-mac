## 0. Approach

Four phases. Replaces D24's distribution algorithm with **Q8 narrow-column threshold lock-in + CSS §3.9 slack-proportional** for the flex pool. Same implementation surface (`Sources/Editor/Renderer/Tables/`), same harness, same wiring; the changes concentrate on:

1. **Token-split spike + measurement extension.** Validate Q1 (ASCII-punctuation as soft-break approximates TextKit's break opportunities), then extend `TableNaturalWidthCache` to hold a `(min, max)` tuple per column.
2. **Algorithm rewrite.** Replace `TableColumnDistribution.distribute(...)` body with the Q8 pre-lock + slack-proportional path; rewrite the 15 XCTest fixtures with new expected values; add new fixtures for Q8 cases.
3. **Builder + harness integration.** Wire the new measurement and distribution through `TK1TableBuilder`. Extend `dump_table_layout` to surface `min`, `max`, `slack`, `q8_locked`, regime. Smoke-test live editor against the regression case + the canonical Decision-log fixture.
4. **Close-out.** Update the D24 manual test plan with new numbers; write D24.2 COMPLETE doc; mark the regression case Fixed; roadmap entry.

Each phase ends in a commit. Stop and surface a `**Question:**` if a phase reveals scope drift (e.g., the Q1 spike shows TextKit's break table diverges materially, or Q8's 120pt threshold is wrong for dogfooded shapes).

## Phase 2 — Slack-proportional distribution + unit-test rewrite

**Goal:** Replace `TableColumnDistribution.distribute(...)` body with the CSS §3.9 algorithm. Rewrite all 15 XCTest fixtures with new expected values. Pure function; behavior change still gated on phase 3's wiring.

**Files updated:**

- `Sources/Editor/Renderer/Tables/TableColumnDistribution.swift`:
  - `distribute` signature changes to `distribute(measurements: [ColumnContentMeasurement], viewportWidth:, minWidthFloor:, narrowThreshold:) -> [CGFloat]`. Default `narrowThreshold = 120`.
  - Body implements:
    - `sum_max ≤ viewport` → return `measurements.map { $0.max }`
    - `sum_min ≥ viewport` (overflow regime) → return `measurements.map { $0.min }`
    - else → Q8 pre-lock iteration (lock cols where `max ≤ narrowThreshold` and fits remaining pool); then slack-proportional on flex cols (`min + slack/total_slack * surplus`, degenerate guard for total_slack=0)
    - Post-pass floor clamp at `minWidthFloor`
  - Old lock-in / proportional-natural code removed.
- `UnitTests/TableColumnDistributionTests.swift`:
  - All 15 tests rewritten with new expected values. Each test gets a `// Algorithm trace:` comment showing the expected math (sum_min, sum_max, Q8 locks, surplus, slack distribution per col). This makes it easy to verify by hand.
  - New test: `testNarrowDataColumnLocksAtMaxViaQ8` — covers the regression case directly. measurements = [(36, 86.5), (70, 1090), (28, 28)]; viewport=250; expected [86.5, 100.5, 28] (Date locked, Decided locked, Decision flex via slack).
  - New test: `testWideContentColumnFlexesViaSlack` — confirms columns above threshold go through slack distribution, not Q8.
  - New test: `testQ8ThresholdTunable` — distribute with narrowThreshold=200 vs 60; same inputs, different lock decisions.
  - New test: `testQ8IteratesUntilFixpoint` — 4 narrow cols at viewport that can't fit all 4 at max simultaneously; confirms Q8 locks as many as fit and the rest go to slack.
  - Updated test: `testSumInvariant_acrossViewportSweep` regime detection now `fits / Q8-lock + slack / slack-only / overflow`.

**DOD:**

- All 15 (now 16+) XCTest fixtures pass.
- Distribution is deterministic; same inputs → same outputs.
- No NaN, Inf, negative widths.
- Sum-of-widths invariants hold per regime.
- Behavior change still gated on phase 3 (`computeColumnWidths` not yet calling new signature).

**Commit:** `D24.2 phase 2 — slack-proportional distribution + unit-test rewrite`

## Risks / open implementation questions

1. **Q1 token-split fidelity** — phase 1 spike gates this. If TextKit's actual break opportunities diverge from our heuristic on dogfooded content, document the divergence and refine. Spec acknowledges this risk; spike confirms or denies.
2. **Cache memory** — extending entries from `CGFloat` to `(CGFloat, CGFloat)` is +1 word per entry; negligible (a 10-table doc with 4 cols = 40 entries × 8 bytes extra = 320 bytes). No concern.
3. **D24 manual test plan rewrite** — width numbers all change. Risk: stale snapshot expectations cause confused future-readers. Mitigation: phase 4 explicitly refreshes the numbers; the v0.6 → v0.6.2 progression chronicled in COMPLETE.
4. **i02 backlog entry semantics** — D24 closed i02. D24.2 fixes a *D24 algorithm gap* discovered on dogfood. Filing this as i05 keeps the backlog honest. Don't reopen i02.
5. **Performance** — `cellMinContentWidth` adds an O(N) text walk per cell. Negligible for typical cell sizes (<1ms for any cell <1KB).
